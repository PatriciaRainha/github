Está com dificuldades nas plataformas Git, GitHub ou GitPages? Aprenda agora mesmo com nosso tutorial.

PASSO 1

Primeiro de tudo precisamos fazer o download do Git em seu site oficial

Git Config

Utilizaremos logo após abrir o cmd (Tecla windows + r -> Digite "cmd" + enter) para configurar o nome de usuário (git config --global user.name "nome") e email (git config --global user.email "email")

Git Init

Inicializa a ferramenta git no repositório selecionado, criando uma pasta .git

Git Clone

O git clone é usado sobretudo para apontar para um repositório existente e fazer um clone ou cópia deste repositório no novo diretório, em outro local. Para teste, iremos apagar a pasta teste e criá-la com o git clone

Git Add

O comando git add pode ser utilizado para adicionar os arquivos que foram modificados ou adicionados.

Git Commit

Utilizado após o git add, o git commit serve para "salvar" as modificações e liberando o uso do git push que falaremos mais abaixo

Git Pull

Git Push

Se o git pull "puxa" o git push irá "empurrar", ou seja, alterando algum código, será preciso dar um "git push" para que o mesmo suba para o GitHub

Git Checkout

Se você reparou no print anterior, quando demos o git status, ele continua na branch master, para trocar para a nova branch utilizaremos o comando git checkout (nome da branch)

Git Merge

Não podemos ficar com duas branchs para sempre, eventualmente teremos que voltar para branch master, mas como todas as atualizações estão na branch versao2, precisamos voltar para a master, e juntar as duas branchs

Git Status

Por fim, utilizamos o git status para mostrar o que está acontecendo com o nosso git, caso haja itens a serem commitados ele nos avisará
